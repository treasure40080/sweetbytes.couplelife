# Git Commit Message 撰寫規範

為了建立統一且清晰的版本控制歷史，所有的 Git Commit Message 請務必嚴格遵循以下格式規範。

## 格式定義

```text
[<型態>] <Jira單號> <簡短主旨>

- 異動項目一
- 異動項目二
```

🚨 **重點規則：**
1. **開頭必須帶有方括號標籤**（例如：`[feature]`），並且**後方必須緊接一個半形空格**。
2. **接著填寫 Jira 單號**（例如：`PXBOX-26837`），並且**後方必須再接一個半形空格**。
3. **最後填寫實際調整的內容**，請務必**清楚、仔細說明**本次 Commit 的改動重點，並建議以條列式方式呈現。

> ⚠️ 注意：各個區塊（標籤、單號、訊息內容）之間都必須有**一個半形空格**，不可省略或連在一起寫！

## 正確範例 (Best Practice)

```text
[feature] PXBOX-26442 建立動態啟動圖 SA 文件與開發規範

- 建立 PXBOX-26442 動態媒體啟動圖系統分析 (SA) 文件
- 建立 ApiModelSafety 技能規範
- 建立 MemorySafety 技能規範
```

## 型態清單

| 型態 | 使用時機 |
|------|----------|
| `[feature]` | 新功能、功能修改、Harness 設定、文件更新 |
| `[bugfix]` | 錯誤修正 |

## 錯誤範例 (Anti-Pattern)

❌ **沒有空格 (連在一起)**：
`[feature]PXBOX-26372新增弱網自動補發機制`

❌ **缺少 Jira 單號**：
`[feature] 新增弱網自動補發機制`

❌ **使用語義化前綴取代正確格式**：
`[PXBOX-27349] feat: 更換 APP icon` ← 錯誤，不是此專案格式
`[PXBOX-27349] docs: 新增文件` ← 錯誤

❌ **單號打錯或格式不一**：
`[feature] pxbox-26372 新增機制` (單號未大寫)
`[feature]  PXBOX-26372  新增機制` (多餘的空白)
